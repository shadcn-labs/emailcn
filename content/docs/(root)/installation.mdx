## Quick Start

### 1. Install react-email

First, ensure you have react-email installed in your project:

```bash
npm install react-email
# or
pnpm add react-email
```

### 2. Add a Theme

Install a theme to style your emails:

```bash
npx shadcn@latest add https://emailcn.com/r/theme-default.json
```

### 3. Add Components

Install individual components:

```bash
# Add a hero component
npx shadcn@latest add https://emailcn.com/r/hero.json

# Add a footer component
npx shadcn@latest add https://emailcn.com/r/footer.json
```

### 4. Add Blocks

Install complete email templates:

```bash
# Add a welcome email
npx shadcn@latest add https://emailcn.com/r/block-onboarding-default.json

# Add an auth magic link email
npx shadcn@latest add https://emailcn.com/r/block-auth-magic-link-default.json
```

## Usage

After installation, files land under **`emails/`** (themes in **`emails/themes/`**, shared font helpers in **`emails/fonts/`**). Each UI component’s default export is a **full React Email document** (`Html` → `Head` → `Preview` → `Tailwind` → `Body`).

**Standalone** — render the component (it’s a complete email):

```tsx
import { Hero } from "@/emails/hero";

export function WelcomeEmail() {
  return <Hero heading="Welcome!" subheading="Thanks for joining us." />;
}
```

**Compose inside your own template** — import the `*Section` partial (e.g. `HeroSection`, `FooterSection`) so you only have one `<Html>`:

```tsx
import { Body, Container, Head, Html, Preview, Tailwind } from "react-email";
import { FooterSection } from "@/emails/footer";
import { HeroSection } from "@/emails/hero";
import { DefaultFonts } from "@/emails/fonts/default";
import { defaultTheme } from "@/emails/themes/default";

export function WelcomeEmail() {
  return (
    <Html>
      <Head>
        <DefaultFonts />
      </Head>
      <Preview>Welcome</Preview>
      <Tailwind config={defaultTheme}>
        <Body className="bg-background font-sans">
          <Container className="mx-auto max-w-container">
            <HeroSection
              heading="Welcome!"
              subheading="Thanks for joining us."
            />
            <FooterSection companyName="Acme" />
          </Container>
        </Body>
      </Tailwind>
    </Html>
  );
}
```

## Registry Structure

In **this repo**, source lives under **`registry/`**. When you **`npx shadcn add`** from [emailcn.com](https://emailcn.com), installs are written to:

- **`emails/<name>.tsx`** — React Email / MJML UI components and blocks
- **`emails/themes/<theme-name>.ts`** — Tailwind email themes
- **`emails/fonts/default.tsx`** — default Inter stack (`DefaultFonts`), and optional **`emails/fonts/*-fonts.tsx`** presets (e.g. `tech-fonts.tsx`, `skin-fonts.tsx`) matching [Resend’s react-email demos](https://github.com/resend/react-email/tree/canary/apps/demo/emails)

Locally you can import from `@/registry/...` paths instead.
